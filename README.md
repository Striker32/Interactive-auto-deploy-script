Решение представляет собой программно-технический комплекс для развертывания тестовой среды web-сайта в условиях ограниченной сетевой видимости (NAT) на локальных мощностях  
В проекте используется утилита туннелирования Cloudpub с месячным ограничением по трафику в 250 ГБ

Требования:
- 1 ядро ЦП, 1 ГБ оперативной памяти, около 7 ГБ долговременной памяти
- Наличие docker
- Реквизиты базы данных к проекту (если таковая есть), должны быть написаны с использованием переменных окружения ([для незнающих сюда](https://habr.com/ru/companies/gnivc/articles/792082/))
- Аккаунт cloudpub и токен 

Перед запуском необходимо переместить исходный код проекта в соответствующую папку. Решение поддерживает импорт дампа базы данных. Для этого поместите дамп в ту же папку, что и исходный код проекта
Решение поддерживает готовые docker-compose файлы, если ваше приложение состоит из нескольких сервисов. Также поместите данный файл в папку исходного кода, если желаете развернуть приложение со своей конфигурацией

Запустить в папке с проектом команду для Linux:

    docker build -t devtestops-master . && docker run -it --rm      -v /var/run/docker.sock:/var/run/docker.sock      -v ~/emptydir:/project      devtestops-master

для windows:

    docker build -t devtestops-master . ; if ($?) { docker run -it --rm -v /var/run/docker.sock:/var/run/docker.sock -v ${PWD/emptydir}:/project devtestops-master }

После данных действий откроется интерактивный скрипт развертывания.
